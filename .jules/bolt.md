## 2024-03-24 - Zustand Destructuring Issue | **Learning:** In components, destructuring properties directly from `useGameStore()` without selectors subscribes the component to the entire store, causing re-renders on ANY state change (such as frequent `particles` updates). This is a major performance bottleneck for a game with an idle/passive particle generation loop. | **Action:** Always use granular state selectors (e.g., `const particles = useGameStore(state => state.particles)`) or `useShallow` to avoid massive re-renders in UI components.
