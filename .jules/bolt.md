## 2024-05-24 - Zustand Destructuring Causing Re-renders
**Learning:** Destructuring `useGameStore()` in Next.js/React without selectors or `useShallow` subscribes the component to the entire store. In an application with passive/automatic updates (e.g. idle game `particles` increasing), this anti-pattern causes excessive rendering even for static components (like tabs) that do not directly consume the frequently changing state variables.
**Action:** Always use selectors when selecting a single state property (`const nullifierHash = useGameStore(state => state.nullifierHash)`). When destructuring multiple state properties, strictly utilize `import { useShallow } from 'zustand/react/shallow'` wrapped inside the store selector hook to maintain reference equality and prevent unnecessary re-renders.
