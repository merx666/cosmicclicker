## 2024-05-19 - Zustand Subscriptions | **Learning:** Codebase-specific performance anti-pattern: Using useGameStore() without selectors (e.g., destructuring the store directly) subscribes the component to the entire global state, causing massive re-renders on frequent updates like passive particle generation. Always use specific state selectors, particularly when multiple properties are needed. | **Action:** Optimize Zustand store selections by replacing destructuring with `useShallow` from `zustand/react/shallow` to prevent unnecessary re-renders.
