## 2025-05-12 - [Zustand React Component Rendering Optimization]\n**Learning:** In this codebase, the pattern `useGameStore()` without specific selectors is used across many components (AdsTab, VoidClubTab, PremiumTab, RouletteTab, SeasonPassTab, SurveyTab, UpgradesTab, etc.). This acts as a subscription to the entire global state, causing components to re-render whenever ANY value in the global state changes (e.g., passive particle generation ticking every second).\n**Action:** Use `useShallow` from `zustand/react/shallow` alongside explicit selector functions when selecting multiple state properties, or direct single-property selectors, to limit re-renders strictly to changes in the properties that the component actually consumes.
