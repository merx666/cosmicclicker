## 2025-02-13 - Destructuring Zustand Store Anti-Pattern
**Learning:** A codebase-specific performance bottleneck exists when components destructured `useGameStore()` without `useShallow` or specific selectors. Because passive particle generation updates the global state every second, any component that destructured the full store (e.g., Tab components) was being unnecessarily re-rendered every single second, even if the state they cared about hadn't changed.
**Action:** When using Zustand in this repository, always use direct state selectors (`useGameStore(state => state.property)`) for single properties, or `useShallow` from `zustand/react/shallow` when multiple properties must be selected, to prevent massive unnecessary re-render cycles caused by frequent background state updates.
