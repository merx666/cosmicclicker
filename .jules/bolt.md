## 2025-05-18 - Zustand Store Re-renders | **Learning:** Codebase-specific performance anti-pattern: Using `useGameStore()` without selectors subscribes the component to the entire global state, causing massive re-renders on frequent updates like passive particle generation. | **Action:** Always use specific state selectors (`useGameStore(state => state.property)`) or `useShallow` for multiple properties to prevent re-renders on unrelated state changes.
