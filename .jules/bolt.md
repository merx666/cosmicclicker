## 2024-04-13 - Zustand State Selection Optimization\n**Learning:** Components destructuring `useGameStore()` subscribe to the entire global state, causing massive re-renders whenever any state (like `particles` updating per second) changes.\n**Action:** Use specific state selectors (e.g., `state => state.property`) or `useShallow` from `zustand/react/shallow` for multiple properties to limit component subscriptions.
