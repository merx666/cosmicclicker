## 2025-02-18 - Prevent Unnecessary Re-renders via Selectors | **Learning:** In Zustand (^5.0.9), destructuring multiple properties like `const { nullifierHash, particles } = useGameStore()` without using selectors subscribes the component to the entire store state. This anti-pattern causes excessive re-renders, especially when frequently updated state (like `particles`) changes. | **Action:** Always use specific state selectors like `useGameStore((state) => state.nullifierHash)` or `useShallow` from `zustand/react/shallow` when destructuring multiple properties to ensure components only re-render when the specific selected state changes.
